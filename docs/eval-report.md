# 评测报告（M6）

> 30 条多步任务用例，覆盖数据分析 / 创建工单(批准) / 创建工单(拒绝) / 查询跟进 /
> 知识检索 / 多步组合六类。每条用例带可程序化校验的成功判据（工具用对、答案含关键信息、
> 写操作按预期通过或被审批拦截），自动判定，无人工打分主观性。
> AutoApprover 按用例的 approve 字段决定批准或拒绝，双向验证审批闸。

## 指标定义

- **任务完成率**：通过 check 断言的用例占比
- **平均工具调用次数**：效率与规划质量（越少越好，反映拆解是否精准）
- **审批拦截率**：写操作用例中，写操作 100% 经人工审批而非 Agent 自主执行的占比——项目安全特色指标
- **端到端延迟**：P50 / P95（含 LLM 多轮 + 工具调用）

## 结果

| 指标 | 数值 |
|---|---|
| 任务完成率 | **29/30 = 97%** |
| 平均工具调用次数 | **2.23 次/任务** |
| 审批拦截率 | **13/13 = 100%** |
| P50 / P95 延迟 | 14.4s / 22.8s |

### 分类别通过率

| 类别 | 通过/总数 |
|---|---|
| 数据分析 | 9/10 |
| 创建工单（批准） | 5/5 |
| 创建工单（拒绝） | 3/3 |
| 查询跟进 | 5/5 |
| 知识检索 | 2/2 |
| 多步组合 | 5/5 |

## 结论

- **规划质量高**：平均 2.23 次工具调用完成任务，多步组合类（查数据+建工单）也稳定在 4~5 次，没有无意义的反复试探。
- **审批闸 100% 生效**：13 个含写操作的用例，全部先经人工审批再执行；批准的写操作执行、拒绝的写操作被拦截且不落库，双向都正确。这是"Agent 落地企业"的关键安全属性。
- **唯一失败（stat-01）是校验判据过严**：用例要求回答字面包含 "400"，Agent 调用 ticket_stats 拿到了正确总数，但回答表述未命中该字面串。属于评测脚本的字符串匹配脆弱性，非 Agent 能力缺陷——如实保留、不放宽判据凑满分（做评测 ≠ 只报好消息）。
